## 一、Agent 执行模式：白盒工具链可视化

![Agent 执行模式](docs/screenshots/06-agent-chain.png)

**适用场景**：你想看 AI 每一步做了什么，追求透明可控。

**交互流程**：

1. **输入需求**：在底部输入框用自然语言描述出行想法，例如：  
   *"今天下午想和3个朋友出去玩几小时，一共4个人2男2女，在郑州，想吃好吃的顺便逛逛，帮我安排"*

2. **智能分析**：系统自动识别场景为「朋友出游」，分析预算等级、出行方式、偏好标签。

3. **计划制定（白盒展示）**：  
   Agent 在左侧控制台实时展示思考过程：
   - 分析需求 → 识别为「游客场景 | 预算Lv2 | 公交」
   - 制定工具调用计划：共 6 个步骤
   - 计划明细：`search_pois → search_pois → check_queue → book_table → check_queue → book_table`

4. **并行执行（带进度反馈）**：  
   每个 Tool 的执行结果实时回显：
   | 步骤 | 工具 | 结果摘要 |
   |------|------|----------|
   | 1 | 搜索景点 | 找到 7 个结果（郑州植物园、郑州动物园等） |
   | 2 | 搜索美食 | 找到 10 个结果（合记烩面、巴奴毛肚火锅等） |
   | 3 | 查询排队 | 合记烩面(总店)：排队约 50 分钟 |
   | 4 | 预订餐厅 | 预订成功！确认码 015A3 |

5. **地图联动**：右侧地图自动标记 POI 位置，绘制推荐路线，数字标注游览顺序。

**核心价值**：用户不仅能得到结果，还能看到 Agent 每一步的决策依据和工具调用详情，真正"白盒化"。

## 二、AI 对话模式：自然语言交互

![AI 对话模式](docs/screenshots/07-ai-chat.png)

**适用场景**：你更喜欢聊天式的交互，像和朋友讨论行程一样。

**交互流程**：

1. **切换模式**：点击顶部「AI对话模式」Tab，进入聊天界面。

2. **发送需求**：同样的自然语言输入，系统以对话气泡形式展示。

3. **AI 回复完整方案**：  
   AI 返回结构化的行程规划，包含：
   - **上午准备**：建议出发前的准备工作
   - **下午行程**：按时间轴排列
     - 美食探索（12:00-13:30）：推荐二七广场附近美食街，郑州特色（烩面、胡辣汤）
     - 逛街购物（13:30-14:30）：步行至二七广场购物中心
     - 休闲时光（14:30-15:30）：世纪欢乐园或郑州动物园

4. **右侧地图同步**：聊天过程中，地图始终展示当前推荐路线的空间分布。

5. **持续对话**：可以接着问"换个安静点的地方"或"预算控制在200以内"，AI 会基于上下文重新规划。

**核心价值**：降低使用门槛，用户无需学习任何操作，像发微信一样说句话就能得到专业规划。

## 三、快速模式：表单化精准规划

![快速模式](docs/screenshots/08-quick-mode.png)

**适用场景**：你对自己的需求很明确，想快速获得精准方案。

**交互流程**：

1. **切换模式**：点击顶部「快速模式」Tab。

2. **参数微调**：左侧表单提供多维度控制：
   - **可用时间**：滑动条调节（如 14:00 → 18:00，系统提示"可用时间约4小时，时间适中"）
   - **偏好权重**：省钱、效率、就近三个维度拖拽调节（0-100）
   - **出行描述**：自然语言补充（与 Agent 模式共用输入）
   - **出行方式**：自动识别 / 公交 / 地铁 / 自驾 / 骑行 / 步行
   - **预算档位**：自动识别 / 经济 / 标准 / 品质 / 奢华
   - **偏好标签**：拍照、夜景、实惠、亲子、历史文化、网红打卡、本地特色、安静
   - **限定区域**：下拉选择行政区划

3. **一键生成**：点击「规划」按钮，系统在秒级内返回结果。

4. **推荐路线方案卡片**：右侧弹出方案面板，包含：
   - 路线标签：「个性推荐路线」「高效打卡路线」「口碑优先路线」
   - AI 推荐语：自然语言描述整条路线的亮点
   - 景点列表：按顺序排列，带评分、区域、人均消费
   - 行程概要：预估花费、总时长、点位数
   - 拖拽排序：可手动调整景点顺序，AI 实时重算路线

**核心价值**：把"模糊需求"和"精准控制"结合——既能自然语言输入，又能通过表单精确调节偏好。

## 四、RAG 消融实验：可解释性验证

![RAG 消融实验](docs/screenshots/09-rag-experiment.png)

**适用场景**：你想理解"为什么推荐这些 POI"，验证系统是否基于真实数据而非幻觉。

**交互流程**：

1. **进入实验面板**：点击顶部「RAG实验」按钮，弹出对比实验界面。

2. **输入测试查询**：例如 *"推荐郑州最好吃的火锅店"*，点击「对比」。

3. **左右对比展示**：

   | 维度 | 无 RAG（纯 LLM） | 有 RAG（检索增强） |
   |------|------------------|--------------------|
   | 推荐结果 | 郑州海底捞、德庄火锅、蜀九香... | 合记烩面(总店)、郑州烤鸭总店... |
   | 数据来源 | 训练语料（可能过时/不存在） | 真实 POI 数据库 |
   | 准确性 | ❌ 推荐大量火锅店（数据中没有火锅店信息） | ✅ 如实反馈"未找到火锅店信息" |
   | 置信度 | 无 | 显示匹配度百分比（郑州动物园 28%、肯德基 20%...） |

4. **实验结论区**：系统自动总结：
   - 检索到 5 个相关 POI（jieba 分词 + TF-IDF + 余弦相似度）
   - 无 RAG：LLM 仅凭训练数据回答，可能推荐不存在的餐厅或景点
   - 有 RAG：先做向量检索再注入 LLM prompt，推荐结果基于真实 POI
   - **RAG 的核心价值在于让大模型"看见"本地数据，从根本上消除推荐场景中的幻觉问题。**

5. **检索过程白盒日志**：可展开查看完整技术链路：
   - 查询输入 → 中文分词（jieba 分词 → 6 个词）
   - TF-IDF 向量化（sklearn TfidfVectorizer，max_features=200，ngram 1~2）
   - 相似度计算（余弦相似度 × 22 条 POI → Top-5）
   - 匹配明细：每个 POI 的相似度百分比与进度条

**核心价值**：不仅是一个功能，更是一个"可解释性产品特性"——让用户和评审都能直观看到 RAG 如何消除 LLM 幻觉。

## 五、三种模式如何选择？

| 模式 | 适合谁 | 特点 | 响应速度 |
|------|--------|------|----------|
| **Agent 执行** | 技术爱好者、产品经理 | 白盒可视化，每一步工具调用透明 | 中等（LLM + 工具链） |
| **AI 对话** | 普通用户、首次体验 | 零学习成本，像聊天一样自然 | 中等（纯 LLM 对话） |
| **快速模式** | 需求明确的效率用户 | 表单控制精准，秒级出方案 | 最快（规则引擎 + 缓存） |

> 💡 **建议**：第一次用选「AI 对话」感受自然语言交互；想深度了解系统逻辑选「Agent 执行」；日常高频使用选「快速模式」。

## 七、快速体验

如果你还没运行项目：

```bash
cd smart-travel-planner
pip install -r requirements.txt
python -m uvicorn backend.main:app --host 0.0.0.0 --port 8000
```

打开浏览器访问 `http://localhost:8000`，即可体验上述全部交互。
